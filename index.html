// DevOps · Automação · Entrega Contínua

O que é
CI/CD?

Integração Contínua e Entrega Contínua são práticas que automatizam o ciclo de desenvolvimento — do commit ao deploy — com velocidade, segurança e confiança.

// fluxo

Pipeline de Entrega

📝
Commit
Dev envia código para o repositório (Git)
✓ passed
🔍
Build
Compilação e análise de dependências
✓ passed
🧪
Testes
Unit, integração e testes e2e automatizados
⟳ running
🔒
Segurança
SAST, DAST e varredura de vulnerabilidades
– pending
📦
Artefato
Docker image ou pacote pronto para deploy
– pending
🚀
Deploy
Push automático para staging / produção
– pending
// conceitos

Fundamentos Essenciais

🔁
Integração Contínua (CI)
Merge frequente de código no repositório principal. Builds e testes automáticos detectam erros cedo.
📤
Entrega Contínua (CD)
Software sempre pronto para deploy em produção após passar por todos os estágios do pipeline.
⚡
Deploy Contínuo
Toda mudança que passa nos testes vai direto para produção — sem aprovação manual.
🐳
Containers
Docker e Kubernetes garantem ambientes consistentes e isolados do dev até a prod.
🧬
Infrastructure as Code
Infraestrutura definida em código (Terraform, Ansible) versionada junto com a aplicação.
📊
Observabilidade
Logs, métricas e rastreamento distribuído para monitorar saúde do sistema em tempo real.

Pipeline em Ação

github-actions / ci-pipeline.yml
$ git push origin main
→ Trigger: push event on main
→ Runner: ubuntu-22.04

$ npm ci
✓ Dependencies installed (1.2s)

$ npm run lint
✓ Lint passed — 0 errors, 0 warnings

$ npm test -- --coverage
✓ 24 tests passed
⚠ Coverage: 87% (threshold: 80%)

$ docker build -t myapp:v1.4.2 .
✓ Image built & pushed to registry

$ kubectl rollout deploy/myapp
✓ Deployment successful → staging
→ URL: https://staging.myapp.dev
